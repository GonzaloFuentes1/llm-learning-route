MLOps: W&B + SLURM

Experiment tracking con Weights & Biases y gestión de jobs de entrenamiento con SLURM. Cubre logging de métricas, artefactos y scripts para runs individuales y en paralelo.

1. Setup de Weights & Biases

2. Logging de métricas durante training

python — dentro del training loop
import math
import wandb

# En cada step (o cada N steps para no sobrecargar)
if step % 10 == 0:
    wandb.log({
        # Loss y perplexidad
        "train/loss": loss.item(),
        "train/perplexity": math.exp(min(loss.item(), 20)),  # clipear para evitar overflow

        # Learning rate
        "train/lr": scheduler.get_last_lr()[0],

        # Gradient norm (útil para detectar spikes)
        "train/grad_norm": grad_norm.item(),

        # Throughput
        "train/tokens_per_sec": tokens_this_step / elapsed_time,

        # Paso global y tokens vistos
        "train/step": global_step,
        "train/tokens_seen": global_step * batch_size * max_seq_len * grad_accum_steps,
    }, step=global_step)

# Métricas de validación (cada 500 steps)
if step % 500 == 0:
    val_loss = evaluate_on_validation_set(model, val_dataloader)
    wandb.log({
        "val/loss": val_loss,
        "val/perplexity": math.exp(min(val_loss, 20)),
    }, step=global_step)
Métricas clave a monitorear: train/loss (debe bajar), train/grad_norm (debe ser estable ~0.5-2.0), val/loss (para detectar overfitting), train/tokens_per_sec (para medir eficiencia del cluster).

3. Logging de artefactos

python — guardar checkpoints como artefactos
import wandb

# Guardar un checkpoint como artefacto W&B
def save_checkpoint_artifact(checkpoint_dir: str, step: int):
    artifact = wandb.Artifact(
        name=f"checkpoint-step-{step}",
        type="model",
        metadata={"step": step, "val_loss": val_loss},
    )
    artifact.add_dir(checkpoint_dir)
    wandb.log_artifact(artifact)


# Guardar distribución de tokens del dataset
def log_token_distribution(token_counts: dict):
    table = wandb.Table(
        columns=["domain", "tokens_B"],
        data=[[k, v / 1e9] for k, v in token_counts.items()]
    )
    wandb.log({"data/token_distribution": wandb.plot.bar(
        table, "domain", "tokens_B", title="Token distribution by domain"
    )})

4. Scripts SLURM para jobs de training

Job único (4 GPUs, 24 horas)
bash — jobs/train_7b.sh
#!/bin/bash
#SBATCH --job-name=lgpt-train-7b
#SBATCH --partition=gpu
#SBATCH --gres=gpu:4
#SBATCH --ntasks-per-node=1
#SBATCH --cpus-per-task=32
#SBATCH --mem=256G
#SBATCH --time=24:00:00
#SBATCH --output=logs/train_%j.out
#SBATCH --error=logs/train_%j.err
#SBATCH --mail-type=END,FAIL
#SBATCH --mail-user=tu@email.com

# Environment
source ~/.bashrc
conda activate latamgpt
export WANDB_PROJECT="latamgpt"
export WANDB_RUN_NAME="pretraining-7b-${SLURM_JOB_ID}"

# Para multi-GPU con accelerate
export CUDA_VISIBLE_DEVICES=0,1,2,3
export OMP_NUM_THREADS=8
export TOKENIZERS_PARALLELISM=false

echo "Job ID: ${SLURM_JOB_ID}"
echo "Nodo: $(hostname)"
echo "GPUs: ${CUDA_VISIBLE_DEVICES}"
echo "Inicio: $(date)"

# Lanzar training
accelerate launch \
  --config_file configs/deepspeed_zero2.yaml \
  --num_processes 4 \
  train.py \
  --config configs/pretraining_7b.yaml \
  --resume_from_checkpoint checkpoints/latest

echo "Fin: $(date)"
Array jobs (múltiples experimentos en paralelo)
bash — jobs/sweep.sh
#!/bin/bash
#SBATCH --job-name=lgpt-sweep
#SBATCH --partition=gpu
#SBATCH --gres=gpu:1
#SBATCH --cpus-per-task=8
#SBATCH --mem=64G
#SBATCH --time=06:00:00
#SBATCH --output=logs/sweep_%A_%a.out
#SBATCH --array=0-7        # 8 jobs en paralelo

# Definir hiperparámetros por índice
LRS=(1e-4 2e-4 3e-4 5e-4 1e-3 2e-3 3e-3 5e-3)
LR=${LRS[$SLURM_ARRAY_TASK_ID]}

source ~/.bashrc
conda activate latamgpt

python finetune.py \
  --learning_rate ${LR} \
  --run_name "sweep-lr-${LR}-${SLURM_ARRAY_TASK_ID}"
bash — enviar y monitorear
# Enviar el job
sbatch jobs/train_7b.sh

# Ver estado de todos mis jobs
squeue -u $USER --format="%.18i %.9P %.20j %.8u %.8T %.10M %.9l %.6D %R"

# Seguir logs en tiempo real
tail -f logs/train_12345.out

# Cancelar si algo está mal
scancel 12345

# Ver cuánto tiempo/recursos usó un job completado
sacct -j 12345 --format=JobID,Elapsed,MaxRSS,MaxVMSize,CPUTime

5. Comparar runs en W&B

python — descargar y comparar runs programáticamente
import wandb
import pandas as pd

api = wandb.Api()

# Obtener todos los runs del proyecto
runs = api.runs("tu-usuario/latamgpt", filters={"tags": {"$in": ["pretraining"]}})

# Crear tabla comparativa
data = []
for run in runs:
    data.append({
        "name": run.name,
        "status": run.state,
        "val_loss": run.summary.get("val/loss", None),
        "final_lr": run.config.get("learning_rate"),
        "tokens_seen": run.summary.get("train/tokens_seen", 0) / 1e9,
    })

df = pd.DataFrame(data).sort_values("val_loss")
print(df.to_string(index=False))
