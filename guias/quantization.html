Quantización para inferencia eficiente
GPTQ, AWQ y GGUF: cuándo usar cada uno, cómo quantizar modelos LatamGPT y cuánta calidad se pierde con cada método.
1. Comparación de métodos
MétodoApproachHardwareIntegraciónCalidad
BitsAndBytes (NF4)Quantización en tiempo de cargaGPU CUDAtransformers nativoMuy buena (training)
GPTQPost-training, calibración offlineGPU CUDAAutoGPTQ / transformersBuena
AWQActivation-aware, calibración offlineGPU CUDAAutoAWQ / vLLMExcelente
GGUF / llama.cppQuantización flexible (Q2-Q8)CPU / Metal / CUDAllama.cpp / OllamaDepende del nivel Q

Regla de elección: AWQ para producción en GPU (mejor throughput en vLLM), GGUF para desarrollo local o hardware sin GPU, BitsAndBytes para fine-tuning (no para inferencia en producción), GPTQ si AWQ no está disponible para tu modelo.

2. AWQ: Activation-Aware Weight Quantization

AWQ identifica los pesos más importantes para la activación y los protege durante la quantización. Resultado: menor degradación que GPTQ al mismo número de bits.

bash
pip install autoawq
python
from awq import AutoAWQForCausalLM
from transformers import AutoTokenizer

MODEL_PATH = "latamgpt/llama3-8b-sft-v1"   # tu modelo fino-tuneado

# 1. Cargar modelo y tokenizador
model = AutoAWQForCausalLM.from_pretrained(MODEL_PATH, safetensors=True)
tokenizer = AutoTokenizer.from_pretrained(MODEL_PATH)

# 2. Dataset de calibración: textos representativos del uso final
calib_data = [
    "Hola, ¿cómo puedo ayudarte hoy?",
    "El modelo de lenguaje grande es una red neuronal que...",
    "Para calcular la derivada de x², aplicamos la regla de la potencia...",
    # añadir ~100-200 ejemplos representativos en es/pt
]

# 3. Configurar quantización
quant_config = {
    "zero_point": True,
    "q_group_size": 128,   # granularidad de los grupos
    "w_bit": 4,            # 4 bits por peso
    "version": "GEMM",     # GEMM = más rápido en batch grande; GEMV = mejor en batch=1
}

# 4. Quantizar (requiere GPU, ~30 min para 8B)
model.quantize(tokenizer, quant_config=quant_config, calib_data=calib_data)

# 5. Guardar
model.save_quantized("latamgpt-llama3-8b-awq-4bit")
tokenizer.save_pretrained("latamgpt-llama3-8b-awq-4bit")
Servir con vLLM (producción)
bash
# vLLM detecta AWQ automáticamente por los metadatos del modelo
python -m vllm.entrypoints.openai.api_server \
  --model latamgpt-llama3-8b-awq-4bit \
  --quantization awq \
  --dtype bfloat16 \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.9
Memoria AWQ 4-bit: Un modelo de 8B en bfloat16 ocupa ~16 GB; en AWQ-4bit ocupa ~5-6 GB. Permite servir LLaMA 3 8B en una GPU de 8 GB como la RTX 3070.
3. GPTQ con transformers

GPTQ está integrado directamente en transformers desde la versión 4.35. No requiere librería externa para cuantizar modelos ya publicados en HF Hub.

bash
pip install optimum auto-gptq
python
from transformers import AutoModelForCausalLM, AutoTokenizer, GPTQConfig

tokenizer = AutoTokenizer.from_pretrained("latamgpt/llama3-8b-sft-v1")

# Dataset de calibración (mínimo 128 ejemplos)
calib_dataset = [
    tokenizer(text, return_tensors="pt").input_ids
    for text in calib_data[:128]
]

gptq_config = GPTQConfig(
    bits=4,
    dataset=calib_dataset,
    group_size=128,
    desc_act=True,   # activar para mejor calidad (más lento)
)

model = AutoModelForCausalLM.from_pretrained(
    "latamgpt/llama3-8b-sft-v1",
    quantization_config=gptq_config,
    device_map="auto",
)

model.save_pretrained("latamgpt-llama3-8b-gptq-4bit")
tokenizer.save_pretrained("latamgpt-llama3-8b-gptq-4bit")

# Subir a HF Hub
model.push_to_hub("latamgpt/llama3-8b-gptq-4bit")
tokenizer.push_to_hub("latamgpt/llama3-8b-gptq-4bit")
Cargar un modelo GPTQ ya quantizado

4. GGUF con llama.cpp (CPU / hardware limitado)

GGUF es el formato de llama.cpp. Permite correr modelos cuantizados en CPU, Apple Silicon (Metal) y GPU con CUDA. Ideal para desarrollo local y prototipado sin GPU dedicada.

Convertir a GGUF
bash
# 1. Clonar llama.cpp y compilar
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
cmake -B build -DGGML_CUDA=ON    # añadir -DGGML_METAL=ON para Mac
cmake --build build --config Release -j

pip install -r requirements.txt

# 2. Convertir modelo HuggingFace a GGUF (F16 = sin quantizar)
python convert_hf_to_gguf.py latamgpt-llama3-8b-sft-v1 \
  --outtype f16 \
  --outfile models/llama3-8b-f16.gguf

# 3. Quantizar el GGUF
./build/bin/llama-quantize \
  models/llama3-8b-f16.gguf \
  models/llama3-8b-Q4_K_M.gguf \
  Q4_K_M

5. Benchmarking: calidad vs rendimiento
python
import time
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

def benchmark_throughput(model, tokenizer, prompt: str, n_tokens: int = 200) -> dict:
    """Mide tokens/segundo y VRAM usada."""
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)

    # Warmup
    with torch.no_grad():
        model.generate(**inputs, max_new_tokens=10)

    # Medición
    torch.cuda.synchronize()
    t0 = time.perf_counter()
    with torch.no_grad():
        out = model.generate(**inputs, max_new_tokens=n_tokens, do_sample=False)
    torch.cuda.synchronize()
    t1 = time.perf_counter()

    n_generated = out.shape[1] - inputs["input_ids"].shape[1]
    tps = n_generated / (t1 - t0)

    vram_mb = torch.cuda.max_memory_allocated() / 1e6

    return {"tokens_per_sec": tps, "vram_mb": vram_mb, "n_tokens": n_generated}

# Comparar
PROMPT = "Explica en detalle qué es el pretraining de modelos de lenguaje y por qué es importante."
models_to_test = {
    "BF16 original":   "latamgpt/llama3-8b-sft-v1",
    "AWQ 4-bit":       "latamgpt/llama3-8b-awq-4bit",
    "GPTQ 4-bit":      "latamgpt/llama3-8b-gptq-4bit",
}

for name, path in models_to_test.items():
    tok = AutoTokenizer.from_pretrained(path)
    m   = AutoModelForCausalLM.from_pretrained(path, device_map="auto", torch_dtype=torch.bfloat16)
    res = benchmark_throughput(m, tok, PROMPT)
    print(f"{name}: {res['tokens_per_sec']:.1f} tok/s, {res['vram_mb']:.0f} MB VRAM")

Resultados típicos en una A100 80GB para LLaMA 3 8B, batch=1:

MétodoThroughputVRAMPPL (WikiText-2)

GPTQ 4-bit~130 tok/s~5.5 GB+0.5 PPL

No confiar en perplexity sola: PPL no captura degradación en tareas de razonamiento o seguimiento de instrucciones. Siempre evaluar el modelo quantizado con lm-eval-harness en los benchmarks del proyecto (MGSM, IFEval) antes de desplegar en producción.
