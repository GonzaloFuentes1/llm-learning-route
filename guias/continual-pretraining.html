Continual Pretraining: de LLaMA a español/portugués

Adapta un modelo base en inglés (LLaMA 3, Mistral) a español y portugués latinoamericano mediante continual pretraining. Cubre mixture de datos, prevención de catastrophic forgetting, configuración de entrenamiento y evaluación intermedia.

1. Continual pretraining vs fine-tuning

Continual PretrainingSFT / Fine-tuning
ObjetivoNext-token prediction (causal LM)Seguir instrucciones / formato de chat
Tokens entrenadosBillones (1B–100B+)Millones (<1B)
Tipo de datosTexto plano, web crawl, librosPares instrucción-respuesta
LR típico1e-4 a 5e-51e-5 a 2e-4 (LoRA)
Cuándo usarloEl modelo base no entiende bien el idioma objetivoEl modelo ya entiende el idioma; solo necesita seguir instrucciones
¿Cuándo hacer continual pretraining? Si el modelo base fue entrenado con <5% de datos en español/portugués (ej. LLaMA 2), necesitas continual pretraining. Si el modelo ya tiene buena comprensión del idioma (ej. LLaMA 3 con 5%+ es/pt), a veces es suficiente con SFT + DPO.

2. Mixture de datos

La clave es no eliminar el inglés del training — eso causa catastrophic forgetting de las capacidades adquiridas en el pretraining original.

FuenteRatio recomendadoPor qué
Español (es-*) — web crawl filtrado35–45%Corpus principal para adaptar el idioma
Portugués (pt-BR) — web crawl filtrado20–25%Cobertura de variante latinoamericana
Inglés — alta calidad (FineWeb-Edu)20–30%Retiene capacidades del modelo original
Código (Python, SQL)5–10%Preserva razonamiento y capacidades técnicas
Datos curados (Wikipedia, libros)5–10%Conocimiento factual de alta densidad

# El mixing se puede hacer durante el training con sample_weights
# o pre-mezclando los datos con proporciones fijas.
# Aquí pre-mezclamos (más reproducible):
import random, json
from pathlib import Path

def stream_source(path: str, weight: float, seed: int = 42):
    """Genera documentos de una fuente con probabilidad proporcional al peso."""
    rng = random.Random(seed)
    for jsonl_file in Path(path).glob("*.jsonl"):
        for line in jsonl_file.open():
            if rng.random() < weight:
                yield json.loads(line)

# Guardar mezcla
with open("data/continual_pretrain_mixture.jsonl", "w") as out:
    for source_name, (path, weight) in SOURCES.items():
        count = 0
        for doc in stream_source(path, weight):
            doc["source"] = source_name
            out.write(json.dumps(doc, ensure_ascii=False) + "\n")
            count += 1
        print(f"{source_name}: {count} documentos")

3. Configuración del entrenamiento

El continual pretraining usa el mismo loop que el pretraining estándar pero con ajustes específicos para evitar forgetting.

python — continual_pretrain.py
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, Trainer
from datasets import load_dataset

BASE_MODEL  = "meta-llama/Llama-3.1-8B"  # modelo base (no-instruct)
OUTPUT_DIR  = "models/llamagpt-es-pt"
DATA_FILE   = "data/continual_pretrain_mixture.jsonl"

tokenizer = AutoTokenizer.from_pretrained(BASE_MODEL)
model = AutoModelForCausalLM.from_pretrained(
    BASE_MODEL,
    torch_dtype=torch.bfloat16,
    attn_implementation="flash_attention_2",  # requiere flash-attn
)

dataset = load_dataset("json", data_files={"train": DATA_FILE}, split="train")

def tokenize(examples):
    return tokenizer(
        examples["text"],
        truncation=True,
        max_length=4096,
        return_overflowing_tokens=True,  # divide documentos largos en chunks
        stride=0,
    )

tokenized = dataset.map(tokenize, batched=True, remove_columns=dataset.column_names)

args = TrainingArguments(
    output_dir=OUTPUT_DIR,
    num_train_epochs=1,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=16,    # effective batch = 64 × num_gpus
    learning_rate=1e-4,                # warmup desde 0; LR menor que pretraining original
    lr_scheduler_type="cosine_with_restarts",
    warmup_steps=500,
    weight_decay=0.1,
    bf16=True,
    gradient_checkpointing=True,
    logging_steps=50,
    save_steps=1000,
    eval_steps=500,
    evaluation_strategy="steps",
    dataloader_num_workers=4,
    report_to="wandb",
)

# DataCollator para CLM (sin padding — usar packing para eficiencia)
from transformers import DataCollatorForLanguageModeling
collator = DataCollatorForLanguageModeling(tokenizer=tokenizer, mlm=False)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=tokenized,
    data_collator=collator,
)
trainer.train(resume_from_checkpoint=True)  # reanuda si hay checkpoint
trainer.save_model(OUTPUT_DIR)
Learning rate crítico: usa un LR de 10–100x menor que el LR original del pretraining. LLaMA 3 fue entrenado con LR pico de 3e-4; para continual pretraining usa 1e-4 a 5e-5. LR muy alto destruye los pesos originales.

5. Evaluación durante el entrenamiento

Evalúa en ambos idiomas (es/pt y en) para detectar forgetting. Monitorea estos benchmarks:

bash — eval_checkpoint.sh
#!/bin/bash
# Correr cada N pasos como SLURM job dependency
CKPT=$1

# Español: perplejidad en Wikipedia es + MGSM
lm_eval --model vllm \

python — compare_eval.py
import json, argparse

parser = argparse.ArgumentParser()
parser.add_argument("--base")
parser.add_argument("--tuned")
args = parser.parse_args()

with open(args.base)  as f: base  = json.load(f)["results"]
with open(args.tuned) as f: tuned = json.load(f)["results"]

print("Benchmark              Base    Tuned   Delta")
print("-" * 50)
for task in base:
    b = base[task].get("acc,none",  base[task].get("acc_norm,none",  0))
    t = tuned[task].get("acc,none", tuned[task].get("acc_norm,none", 0))
    delta = t - b
    flag = "⚠️ " if delta < -0.03 else "  "
    print(f"{flag}{task:22s} {b:.3f}  {t:.3f}  {delta:+.3f}")
Umbral de alarma: si cualquier benchmark en inglés cae más de 3 puntos porcentuales respecto al modelo base, hay forgetting significativo. Aumenta el ratio de replay en inglés y reduce el LR.
