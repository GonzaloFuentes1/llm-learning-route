Entrenamiento de Tokenizadores para es/pt
Entrenar un tokenizador BPE desde cero y extender el vocabulario de LLaMA/Mistral para español y portugués latinoamericano — reduciendo la fertility rate y acelerando el entrenamiento.
1. Por qué el tokenizador de LLaMA es subóptimo para es/pt

El tokenizador de LLaMA 3 fue entrenado principalmente sobre texto en inglés. Como resultado, palabras comunes en español como "educación" o "también" se tokenizan en 3-5 tokens, mientras que en inglés "education" o "also" son 1-2 tokens. Esto tiene consecuencias directas:

MétricaLLaMA 3 tokenizerTokenizador es/pt dedicado
Fertility rate (es)~2.4 tokens/palabra~1.5 tokens/palabra
Tokens para "educación"4 tokens1 token
Contexto efectivo100%~160% (mismo window)
Costo de inferencia100%~65% (menos tokens)

Fertility rate: promedio de tokens por palabra en un corpus de referencia. Menor fertility = tokenizador más eficiente para ese idioma.

¿Siempre hay que cambiar el tokenizador? No. Para fine-tuning sobre pocas épocas, el costo de extender vocabulario (reentrenar embeddings) puede no compensar. Vale la pena cuando: (1) se hace continual pretraining desde cero, (2) el vocabulario de inglés tiene menos del 30% de overlap con el corpus objetivo, (3) el presupuesto de inference es una restricción dura.
2. Preparar corpus de entrenamiento

Para entrenar un buen tokenizador necesitamos texto representativo del dominio objetivo: web en español/portugués, documentos técnicos, y algo de código. Un corpus de ~10GB es suficiente para converger.

python
from datasets import load_dataset
from itertools import islice

def get_training_corpus(batch_size: int = 1000):
    """Iterador perezoso sobre corpus multilingüe es/pt/en."""

    # CulturaX: web crawl limpio en es y pt
    es_ds = load_dataset("uonlp/CulturaX", "es", split="train", streaming=True)
    pt_ds = load_dataset("uonlp/CulturaX", "pt", split="train", streaming=True)

    # FineWeb: texto web en inglés de calidad (para mantener capacidad en inglés)
    en_ds = load_dataset("HuggingFaceFW/fineweb", "sample-10BT",
                         split="train", streaming=True)

    # Intercalar en ratio 40% es, 30% pt, 30% en
    sources = [
        (es_ds, 0.40),
        (pt_ds, 0.30),
        (en_ds, 0.30),
    ]

    iterators = [(iter(ds), weight) for ds, weight in sources]

    while True:
        batch = []
        for it, weight in iterators:
            n = max(1, int(batch_size * weight))
            for ex in islice(it, n):
                batch.append(ex["text"])
        if not batch:
            break
        yield batch
3. Entrenar BPE desde cero

Usamos la librería tokenizers de HuggingFace para entrenar un tokenizador BPE (Byte-Pair Encoding) optimizado para español y portugués.

python
from tokenizers import Tokenizer
from tokenizers.models import BPE
from tokenizers.trainers import BpeTrainer
from tokenizers.pre_tokenizers import ByteLevel
from tokenizers.normalizers import NFC
from tokenizers.processors import TemplateProcessing

# Configurar el tokenizador base
tokenizer = Tokenizer(BPE(unk_token="<unk>"))
tokenizer.normalizer = NFC()           # normalizar Unicode (importante para acentos)
tokenizer.pre_tokenizer = ByteLevel(add_prefix_space=False)

# Tokens especiales
SPECIAL_TOKENS = [
    "<unk>", "<s>", "</s>", "<pad>",
    # Chat template tokens
    "<|user|>", "<|assistant|>", "<|system|>", "<|end|>",
]

trainer = BpeTrainer(
    vocab_size=65_536,           # 64k tokens (LLaMA 3 usa 128k)
    special_tokens=SPECIAL_TOKENS,
    min_frequency=2,             # ignorar n-gramas con <2 apariciones
    show_progress=True,
)

# Entrenar — puede tomar 1-3 horas sobre 10GB de texto
tokenizer.train_from_iterator(get_training_corpus(), trainer=trainer)

# Post-processor para añadir BOS/EOS automáticamente
tokenizer.post_processor = TemplateProcessing(
    single="<s> $A </s>",
    pair="<s> $A </s> $B:1 </s>:1",
    special_tokens=[("<s>", tokenizer.token_to_id("<s>")),
                    ("</s>", tokenizer.token_to_id("</s>"))],
)

tokenizer.save("tokenizer-latamgpt-65k.json")
print(f"Vocabulary size: {tokenizer.get_vocab_size()}")

4. Extender el tokenizador de LLaMA 3 (opción recomendada)

En lugar de entrenar desde cero, podemos añadir nuevos tokens BPE al tokenizador de LLaMA 3. Esto preserva compatibilidad con pesos preentrenados y solo requiere inicializar embeddings para los tokens nuevos.

python
from tokenizers import Tokenizer
from tokenizers.models import BPE
from tokenizers.trainers import BpeTrainer
from tokenizers.pre_tokenizers import ByteLevel
from transformers import AutoTokenizer

BASE_TOKENIZER = "meta-llama/Meta-Llama-3.1-8B"

# 1. Entrenar un tokenizador auxiliar SOLO sobre texto en es/pt
aux_tokenizer = Tokenizer(BPE(unk_token="<unk>"))
aux_tokenizer.pre_tokenizer = ByteLevel(add_prefix_space=False)

aux_trainer = BpeTrainer(
    vocab_size=8_000,   # 8k tokens nuevos a agregar
    min_frequency=5,
    show_progress=True,
)
aux_tokenizer.train_from_iterator(get_training_corpus(), aux_trainer)

# 2. Extraer los tokens nuevos (los que no existen en LLaMA)
llama_tok = AutoTokenizer.from_pretrained(BASE_TOKENIZER)
llama_vocab = set(llama_tok.get_vocab().keys())

new_tokens = [
    tok for tok in aux_tokenizer.get_vocab().keys()
    if tok not in llama_vocab
]
print(f"Tokens nuevos a añadir: {len(new_tokens)}")  # ~5,000-7,000

# 3. Añadir al tokenizador base
llama_tok.add_tokens(new_tokens)
llama_tok.save_pretrained("llama3-extended-tokenizer")

print(f"Vocabulario original: 128,256")
print(f"Vocabulario extendido: {len(llama_tok)}")
5. Inicializar embeddings para nuevos tokens

Los nuevos tokens necesitan embeddings iniciales. La mejor estrategia es inicializarlos como el promedio de los embeddings de los subwords con los que se tokenizaba la misma palabra antes de la extensión.

python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

BASE_MODEL = "meta-llama/Meta-Llama-3.1-8B"

orig_tok = AutoTokenizer.from_pretrained(BASE_MODEL)
ext_tok  = AutoTokenizer.from_pretrained("llama3-extended-tokenizer")

model = AutoModelForCausalLM.from_pretrained(
    BASE_MODEL, torch_dtype=torch.bfloat16
)

# Expandir la capa de embeddings
n_orig = len(orig_tok)   # 128,256
n_new  = len(ext_tok)    # 128,256 + nuevos tokens
model.resize_token_embeddings(n_new)

embed_weight = model.get_input_embeddings().weight  # shape: [vocab_size, hidden]

def init_new_token_embedding(new_token: str) -> torch.Tensor:
    """Inicializa el embedding como promedio de subwords en el tokenizador original."""
    subword_ids = orig_tok.encode(new_token, add_special_tokens=False)
    if not subword_ids:
        return embed_weight[:n_orig].mean(dim=0)
    return embed_weight[subword_ids].mean(dim=0)

# Inicializar solo los tokens nuevos
with torch.no_grad():
    for new_token in new_tokens:
        new_id = ext_tok.convert_tokens_to_ids(new_token)
        if new_id >= n_orig:  # es realmente un token nuevo
            embed_weight[new_id] = init_new_token_embedding(new_token)

# También inicializar la capa lm_head (output embeddings)
if hasattr(model, "lm_head") and model.lm_head.weight is not model.get_input_embeddings().weight:
    lm_head_weight = model.lm_head.weight
    model.resize_token_embeddings(n_new)  # ya lo hicimos, pero por si acaso
    # Copiar los mismos valores inicializados
    with torch.no_grad():
        lm_head_weight[n_orig:] = embed_weight[n_orig:]

model.save_pretrained("llama3-extended-init")
ext_tok.save_pretrained("llama3-extended-init")
Importante: Los nuevos embeddings necesitan entrenarse. El modelo con vocabulario extendido debe pasar por continual pretraining antes de usarse para inferencia. Los embeddings inicializados por promedio son un buen punto de partida, pero el modelo necesita aprender a usarlos en contexto.
6. Evaluar la calidad del tokenizador
python
from transformers import AutoTokenizer
from datasets import load_dataset

def compute_fertility_rate(tokenizer, texts: list[str]) -> float:
    """Tokens generados / palabras totales. Menor = más eficiente."""
    total_tokens = 0
    total_words  = 0
    for text in texts:
        tokens = tokenizer.encode(text, add_special_tokens=False)
        words  = text.split()
        total_tokens += len(tokens)
        total_words  += len(words)
    return total_tokens / total_words if total_words > 0 else 0

# Preparar textos de evaluación
eval_texts = {
    "es": load_dataset("uonlp/CulturaX", "es", split="train",
                       streaming=True).take(1000),
    "pt": load_dataset("uonlp/CulturaX", "pt", split="train",
                       streaming=True).take(1000),
    "en": load_dataset("HuggingFaceFW/fineweb", "sample-10BT",
                       split="train", streaming=True).take(1000),
}
eval_corpus = {lang: [ex["text"] for ex in ds] for lang, ds in eval_texts.items()}

# Comparar tokenizadores
tokenizers_to_compare = {
    "LLaMA 3 original":   AutoTokenizer.from_pretrained("meta-llama/Meta-Llama-3.1-8B"),
    "LLaMA 3 extendido":  AutoTokenizer.from_pretrained("llama3-extended-init"),
}

print(f"{'Tokenizador':<25} {'es':<8} {'pt':<8} {'en':<8}")
print("-" * 50)
for name, tok in tokenizers_to_compare.items():
    rates = {lang: compute_fertility_rate(tok, texts)
             for lang, texts in eval_corpus.items()}
    print(f"{name:<25} {rates['es']:<8.3f} {rates['pt']:<8.3f} {rates['en']:<8.3f}")

Trade-off: El BPE nativo es el más eficiente para es/pt pero pierde eficiencia en inglés. El extendido ofrece un balance razonable sin romper compatibilidad con los pesos preentrenados. Para LatamGPT, el extendido es la opción más práctica.
7. Continual pretraining con vocabulario extendido

Con el modelo inicializado y el tokenizador extendido, el continual pretraining sigue el mismo flujo que la guía correspondiente con una diferencia: hay que congelar los embeddings de los tokens originales durante las primeras épocas para no degradar el inglés.

python
import torch
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("llama3-extended-init",
                                              torch_dtype=torch.bfloat16)
n_orig = 128_256  # vocabulario original de LLaMA 3

def freeze_original_embeddings(model, n_orig_tokens: int):
    """Congela los embeddings de los tokens originales, deja libres los nuevos."""
    embed = model.get_input_embeddings()

    def make_hook(n_orig):
        def hook(grad):
            grad[:n_orig] = 0  # no actualizar tokens originales
            return grad
        return hook

    embed.weight.register_hook(make_hook(n_orig_tokens))

# Activar durante las primeras ~1000 steps para estabilizar los nuevos embeddings
freeze_original_embeddings(model, n_orig)

# Después de 1000 steps, descongelar para permitir ajuste fino del vocabulario completo
# (simplemente no registrar el hook para las siguientes etapas)
