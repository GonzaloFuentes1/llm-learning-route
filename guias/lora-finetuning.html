LoRA y QLoRA: Fine-tuning eficiente

Adapta modelos decoder grandes (LLaMA, Mistral, Qwen) a tareas en español y portugués sin necesitar el VRAM de un full fine-tune. LoRA congela los pesos base e inyecta matrices de bajo rango; QLoRA agrega cuantización 4-bit al modelo base.

1. LoRA vs QLoRA vs full fine-tune

MétodoVRAM (7B)VelocidadCalidadCuándo usarlo
Full fine-tune~120 GBLentoMáximaPretraining continuo, cambio de dominio profundo
LoRA~16 GBRápidoMuy buenaSFT en bf16, hardware moderno (A100/H100)
QLoRA~6-8 GBMás lento que LoRABuenaGPU consumer (3090, 4090, L4), cluster compartido
Regla práctica: Si tienes A100 80 GB, usa LoRA en bf16 — más rápido que QLoRA y sin pérdida de precisión por cuantización. Si tienes menos de 24 GB de VRAM, usa QLoRA con load_in_4bit=True.

2. Instalación

bash
pip install transformers peft accelerate bitsandbytes trl datasets
bitsandbytes en Linux: funciona out-of-the-box. En Windows usa WSL2 o el cluster. La versión para AMD/ROCm es bitsandbytes-rocm.

3. Fine-tuning con QLoRA (código completo)

Ejemplo de SFT para instrucciones en español usando trl.SFTTrainer. Los datos deben estar en formato conversacional (lista de mensajes).

python — qlora_sft.py
import torch
from datasets import load_dataset
from transformers import AutoTokenizer, AutoModelForCausalLM, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, TaskType
from trl import SFTTrainer, SFTConfig

BASE_MODEL = "meta-llama/Llama-3.2-3B-Instruct"  # cambia según el modelo base
OUTPUT_DIR = "models/lora-es-instruct"

# Cuantización 4-bit para QLoRA
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",        # NF4 > int4 para pesos de distribución normal
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,   # ahorra ~0.4 GB adicional
)

# Cargar modelo y tokenizador
tokenizer = AutoTokenizer.from_pretrained(BASE_MODEL)
tokenizer.pad_token = tokenizer.eos_token  # necesario para batch training
tokenizer.padding_side = "right"

model = AutoModelForCausalLM.from_pretrained(
    BASE_MODEL,
    quantization_config=bnb_config,
    device_map="auto",
    torch_dtype=torch.bfloat16,
)
model.config.use_cache = False  # incompatible con gradient checkpointing

# Configuración LoRA
lora_config = LoraConfig(
    r=16,                     # rank — empieza con 16, sube a 64 si la loss no converge
    lora_alpha=32,            # escala = alpha/r, mantén alpha = 2*r
    target_modules=[          # módulos donde inyectar adaptadores
        "q_proj", "k_proj", "v_proj", "o_proj",
        "gate_proj", "up_proj", "down_proj",
    ],
    lora_dropout=0.05,
    bias="none",
    task_type=TaskType.CAUSAL_LM,
)

# Dataset en formato chat
dataset = load_dataset("json", data_files={"train": "data/sft_es.jsonl"})

# La función formatea cada ejemplo como string con el chat template del modelo
def format_chat(example):
    messages = example["messages"]  # lista de {"role": ..., "content": ...}
    return tokenizer.apply_chat_template(messages, tokenize=False, add_generation_prompt=False)

# Entrenamiento
training_args = SFTConfig(
    output_dir=OUTPUT_DIR,
    num_train_epochs=3,
    per_device_train_batch_size=2,
    gradient_accumulation_steps=8,   # effective batch = 16
    gradient_checkpointing=True,     # ahorra VRAM, 20-30% más lento
    learning_rate=2e-4,
    lr_scheduler_type="cosine",
    warmup_ratio=0.05,
    bf16=True,
    logging_steps=10,
    save_steps=200,
    max_seq_length=2048,
    packing=True,                    # sequence packing para eliminar padding
    report_to="wandb",
)

trainer = SFTTrainer(
    model=model,
    args=training_args,
    train_dataset=dataset["train"],
    peft_config=lora_config,
    formatting_func=format_chat,
)

trainer.train()
trainer.save_model(OUTPUT_DIR)
Formato de datos esperado (sft_es.jsonl): 
{"messages": [{"role": "system", "content": "Eres un asistente útil en español."}, {"role": "user", "content": "¿Qué es la fotosíntesis?"}, {"role": "assistant", "content": "La fotosíntesis es..."}]}

5. Guardar, cargar y mergear adaptador

python — merge_adapter.py
from peft import PeftModel
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

BASE_MODEL = "meta-llama/Llama-3.2-3B-Instruct"
ADAPTER_PATH = "models/lora-es-instruct"
MERGED_PATH  = "models/lora-es-instruct-merged"

# Opción A: cargar con adaptador (para inferencia con peft)
model = AutoModelForCausalLM.from_pretrained(BASE_MODEL, torch_dtype=torch.bfloat16)
model = PeftModel.from_pretrained(model, ADAPTER_PATH)

# Opción B: mergear pesos y guardar modelo standalone
merged = model.merge_and_unload()
merged.save_pretrained(MERGED_PATH)
AutoTokenizer.from_pretrained(BASE_MODEL).save_pretrained(MERGED_PATH)

print("Modelo mergeado guardado en", MERGED_PATH)
¿Cuándo mergear? Mergea antes de subir a HuggingFace Hub o de servir con vLLM — vLLM no carga adaptadores PEFT directamente. Durante desarrollo, usa PeftModel para iterar más rápido sin necesitar espacio extra en disco.

6. Diagnóstico: cuándo LoRA no es suficiente

SíntomaCausa probableSolución
Loss no baja de 1.5 tras 1000 pasosRank muy bajo o LR muy pequeñoSubir r a 64, LR a 3e-4
Modelo genera en inglésChat template incorrecto o datos mixtosVerificar apply_chat_template, filtrar idioma
Respuestas truncadas o repetitivasmax_seq_length corto o packing sin máscaraSubir max_seq_length, revisar attention mask
OOM en forward passBatch demasiado grande para VRAM disponibleReducir per_device_train_batch_size a 1, subir gradient_accumulation_steps
Tarea requiere conocimiento nuevoLoRA no puede inyectar conocimiento factualConsiderar continual pretraining primero
