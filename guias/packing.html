Sequence Packing

Técnica para eliminar el padding en batches de pretraining: combinar múltiples documentos en una sola secuencia de longitud fija. Puede recuperar 30-50% de tokens que de otra forma serían desperdiciados.

1. El problema: padding ineficiente

En pretraining, los documentos tienen longitudes variadas. Para formar batches, el estándar es truncar al máximo y rellenar con tokens de padding los documentos más cortos. Esto desperdicia compute:

ejemplo sin packing
max_seq_len = 2048

Doc A: [tok1, tok2, ..., tok120,  PAD, PAD, ..., PAD]  ← 120 tokens útiles, 1928 padding
Doc B: [tok1, ..., tok2048]                              ← 2048 tokens útiles (raro)
Doc C: [tok1, ..., tok350,  PAD, ..., PAD]              ← 350 tokens útiles, 1698 padding

Eficiencia real: (120 + 2048 + 350) / (3 * 2048) = 40.5%
→ 59.5% del compute se gasta en tokens de padding inútiles

Con packing: combinamos A + C en una sola secuencia de 2048 tokens (120 + 350 + algo más), eliminando casi todo el padding.

2. Greedy bin-packing

El algoritmo más simple: para cada secuencia en construcción, agregar documentos hasta que no quepan más. Separar documentos con el token EOS.

python — packing.py
from typing import Iterator
import numpy as np


def greedy_pack(
    token_sequences: list[list[int]],
    max_seq_len: int,
    eos_token_id: int,
) -> Iterator[list[int]]:
    """
    Combina secuencias de tokens en bloques de max_seq_len.
    Cada documento termina con eos_token_id.
    Descarta el exceso si un doc no cabe (trunca al máximo permitido).
    """
    current: list[int] = []

    for seq in token_sequences:
        # Agregar EOS al final del documento
        seq_with_eos = seq + [eos_token_id]

        # Si el doc solo no cabe en una secuencia, truncar
        if len(seq_with_eos) > max_seq_len:
            seq_with_eos = seq_with_eos[:max_seq_len]

        # Si no cabe en el bloque actual, emitir el bloque y empezar uno nuevo
        if len(current) + len(seq_with_eos) > max_seq_len:
            if current:
                # Rellenar el resto con padding (solo si es necesario)
                padded = current + [0] * (max_seq_len - len(current))
                yield padded
            current = seq_with_eos
        else:
            current.extend(seq_with_eos)

    # Emitir el último bloque si tiene contenido
    if current:
        padded = current + [0] * (max_seq_len - len(current))
        yield padded


# Uso:
all_token_sequences = [...]  # lista de listas de token IDs
packed = list(greedy_pack(all_token_sequences, max_seq_len=2048, eos_token_id=2))

3. Attention mask por documento

Sin una máscara adecuada, los tokens de un documento ven los tokens del documento anterior en la misma secuencia, lo que introduce cross-document attention leakage. Para evitarlo, necesitas una máscara de bloque diagonal.

Flash Attention 2+ soporta nativamente secuencias con múltiples documentos usando el parámetro cu_seqlens (cumulative sequence lengths), que es la forma más eficiente.

python — attention mask por documento
import torch


def build_document_attention_mask(
    doc_lengths: list[int],
    max_seq_len: int,
    dtype=torch.bool,
) -> torch.Tensor:
    """
    Crea una máscara de atención causal que bloquea el cross-document attention.
    Returns: (max_seq_len, max_seq_len) bool mask. True = permitir atención.
    """
    mask = torch.zeros(max_seq_len, max_seq_len, dtype=dtype)
    start = 0
    for length in doc_lengths:
        end = start + length
        # Atención causal dentro del documento (triangulo inferior)
        for i in range(start, min(end, max_seq_len)):
            for j in range(start, min(i + 1, max_seq_len)):
                mask[i, j] = True
        start = end
        if start >= max_seq_len:
            break
    return mask


# Para FlashAttention-2: usar cu_seqlens directamente
def get_cu_seqlens(doc_lengths: list[int]) -> torch.Tensor:
    """Cumulative sequence lengths para flash_attn_varlen_func."""
    cumsum = [0]
    for l in doc_lengths:
        cumsum.append(cumsum[-1] + l)
    return torch.tensor(cumsum, dtype=torch.int32)
En la práctica: La mayoría de frameworks modernos (OLMo, LLaMA) implementan esto internamente. Si usas Flash Attention con el argumento cu_seqlens, el modelo maneja automáticamente el bloqueo entre documentos sin necesidad de construir la máscara explícitamente.

4. Integración con HuggingFace

Si usas el ecosistema HuggingFace, la forma más simple es pre-procesar tu dataset con packing antes del entrenamiento:

python — dataset con packing
from datasets import Dataset
from transformers import AutoTokenizer, DataCollatorForLanguageModeling
import numpy as np


def pack_dataset(
    examples: dict,
    tokenizer,
    max_seq_len: int = 2048,
) -> dict:
    """Función de map para empacar un dataset de HuggingFace."""
    all_tokens = []
    for text in examples["text"]:
        tokens = tokenizer(text, add_special_tokens=False)["input_ids"]
        tokens.append(tokenizer.eos_token_id)
        all_tokens.extend(tokens)

    packed_input_ids = []
    packed_labels = []
    for i in range(0, len(all_tokens) - max_seq_len + 1, max_seq_len):
        chunk = all_tokens[i : i + max_seq_len]
        if len(chunk) == max_seq_len:
            packed_input_ids.append(chunk)
            packed_labels.append(chunk)  # CLM: labels = input_ids

    return {"input_ids": packed_input_ids, "labels": packed_labels}


# Usar en el pipeline de training
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")
raw_dataset = Dataset.from_json("data/raw/es_sample.jsonl")

packed = raw_dataset.map(
    lambda examples: pack_dataset(examples, tokenizer, max_seq_len=2048),
    batched=True,
    batch_size=1000,
    remove_columns=raw_dataset.column_names,
    num_proc=16,
)

collator = DataCollatorForLanguageModeling(tokenizer=tokenizer, mlm=False)
print(f"Sequences packed: {len(packed):,}")

5. Comparativa de eficiencia

python — medir eficiencia
import numpy as np

def measure_packing_efficiency(doc_lengths: list[int], max_seq_len: int) -> dict:
    """Compara tokens útiles con vs sin packing."""
    # Sin packing: cada doc se trunca/rellena a max_seq_len
    total_tokens_without = len(doc_lengths) * max_seq_len
    useful_tokens = sum(min(l, max_seq_len) for l in doc_lengths)
    efficiency_without = useful_tokens / total_tokens_without

    # Con packing: cuántas secuencias llenas obtenemos
    total_tokens_raw = sum(min(l + 1, max_seq_len) for l in doc_lengths)  # +1 para EOS
    n_sequences = total_tokens_raw // max_seq_len
    total_tokens_with = n_sequences * max_seq_len
    efficiency_with = (total_tokens_raw - (total_tokens_with - total_tokens_raw % max_seq_len)) / total_tokens_with

    return {
        "sin_packing": f"{efficiency_without:.1%}",
        "con_packing": f"{min(efficiency_with, 1.0):.1%}",
        "mejora": f"{min(efficiency_with, 1.0) / efficiency_without:.1f}x",
    }

# Simular distribución típica de documentos web en español
rng = np.random.default_rng(42)
doc_lengths = rng.integers(50, 5000, size=100_000).tolist()

resultado = measure_packing_efficiency(doc_lengths, max_seq_len=2048)
print(f"Sin packing: {resultado['sin_packing']}")
print(f"Con packing: {resultado['con_packing']}")
print(f"Mejora:      {resultado['mejora']}")
