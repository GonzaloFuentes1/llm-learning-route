Speculative Decoding: 2-3× más rápido sin perder calidad
Un modelo pequeño (draft) propone tokens, el modelo grande (target) los verifica en paralelo. La distribución de salida es idéntica al modelo original — sin ninguna degradación de calidad.
1. Cómo funciona

La observación clave: el cuello de botella de LLM inference no es la computación, sino la latencia de memoria (leer pesos del modelo en cada paso). Speculative decoding amortiza esto verificando K tokens en un solo forward pass del modelo grande.

El loop básico:

  1. El draft model (pequeño, rápido) genera K tokens autoregressivamente: t₁, t₂, …, tₖ
  2. El target model (grande) recibe el contexto + los K tokens propuestos y calcula probabilidades para K+1 posiciones en un solo forward pass
  3. Se acepta o rechaza cada token propuesto mediante muestreo de rechazo (rejection sampling)
  4. Los tokens aceptados se añaden a la secuencia; el primer rechazado se reemplaza con una muestra del target

Propiedad fundamental: La distribución de tokens generados es matemáticamente idéntica a la del target model solo. No hay trade-off de calidad — solo velocidad.

SituaciónSpeedupPor qué
Draft muy alineado con target (texto repetitivo, código)2-4×Casi todos los tokens se aceptan
Texto creativo / aleatorio1.2-1.8×Muchos rechazos
Batch grande (≥16 requests)~1× o menosEl target ya está saturado; no gana
Speculative decoding es solo útil para batch size pequeño (1-4 requests). En servidores con alto throughput (muchas requests simultáneas) no da ventaja — el modelo target ya está ocupado. Úsalo para latency-sensitive applications, no para throughput máximo.
2. Speculative Decoding con vLLM (producción)

vLLM tiene soporte nativo. Solo hay que especificar el draft model y el número de tokens especulativos por paso.

bash
python -m vllm.entrypoints.openai.api_server \
  --model latamgpt/llama3-8b-sft-v1 \
  --speculative-model latamgpt/llama3-68m-draft \
  --num-speculative-tokens 5 \
  --speculative-draft-tensor-parallel-size 1 \
  --dtype bfloat16 \
  --max-model-len 8192

3. Con transformers (prototipado)
python
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

TARGET_MODEL = "latamgpt/llama3-8b-sft-v1"
DRAFT_MODEL  = "meta-llama/Llama-3.2-1B-Instruct"

target_tok = AutoTokenizer.from_pretrained(TARGET_MODEL)
target = AutoModelForCausalLM.from_pretrained(
    TARGET_MODEL, torch_dtype=torch.bfloat16, device_map="cuda:0"
)

draft = AutoModelForCausalLM.from_pretrained(
    DRAFT_MODEL, torch_dtype=torch.bfloat16, device_map="cuda:0"
)

messages = [{"role": "user", "content": "Explica la atención multi-cabeza."}]
input_ids = target_tok.apply_chat_template(
    messages, add_generation_prompt=True, return_tensors="pt"
).to("cuda")

# Speculative decoding: assistant_model = draft model
output = target.generate(
    input_ids,
    assistant_model=draft,        # ← activa speculative decoding
    max_new_tokens=256,
    do_sample=False,
    temperature=1.0,
)

print(target_tok.decode(output[0][input_ids.shape[1]:], skip_special_tokens=True))
Prompt lookup decoding (sin draft model separado)

Una variante más sencilla: buscar n-gramas en el prompt que coincidan con los últimos tokens y usarlos como "draft". Útil para documentos largos con repeticiones (código, JSON, traducciones).

python
output = target.generate(
    input_ids,
    prompt_lookup_num_tokens=10,  # ← sin draft model, busca en el prompt
    max_new_tokens=256,
    do_sample=False,
)
4. Medusa Heads (sin modelo draft separado)

Medusa añade K cabezas de predicción al propio modelo target, cada una predice el token en la posición t+k. No requiere un segundo modelo, pero sí requiere fine-tuning para entrenar las cabezas.

bash
pip install medusa-llm
python
from medusa.model.medusa_model import MedusaModel

# Cargar un modelo con cabezas Medusa ya entrenadas
# (o usar FasterDecoding/medusa-vicuna-7b-v1.3 como referencia)
model = MedusaModel.from_pretrained(
    "FasterDecoding/medusa-vicuna-7b-v1.3",
    torch_dtype=torch.float16,
    device_map="cuda",
)

# La API de generación es la misma — Medusa opera internamente
output = model.generate(
    input_ids,
    medusa_choices=[[0], [0, 0], [1], [0, 1], [2]],  # árbol de candidatos
    max_new_tokens=256,
    temperature=0.0,
)

Para LatamGPT: si se tiene un checkpoint SFT, se pueden entrenar las cabezas Medusa sobre datos en español/portugués (~1 época, muy rápido) y conseguir 1.5-2× speedup sin cambiar el modelo base.

5. Medir el speedup real
python
import time
import torch

def benchmark_generate(model, tokenizer, prompt: str,
                        n_tokens: int = 200,
                        draft_model=None) -> dict:
    input_ids = tokenizer(prompt, return_tensors="pt").input_ids.to(model.device)

    kwargs = dict(max_new_tokens=n_tokens, do_sample=False)
    if draft_model is not None:
        kwargs["assistant_model"] = draft_model

    # Warmup
    with torch.no_grad():
        model.generate(**input_ids._asdict() if hasattr(input_ids, '_asdict')
                       else {"input_ids": input_ids}, max_new_tokens=10)

    torch.cuda.synchronize()
    t0 = time.perf_counter()
    with torch.no_grad():
        out = model.generate(input_ids=input_ids, **kwargs)
    torch.cuda.synchronize()
    elapsed = time.perf_counter() - t0

    n_gen = out.shape[1] - input_ids.shape[1]
    return {"tok/s": n_gen / elapsed, "elapsed": elapsed, "n_tokens": n_gen}

PROMPT = "Explica detalladamente cómo funciona el algoritmo Transformer, incluyendo la atención multi-cabeza, las capas feed-forward y la normalización de capas."

r_baseline = benchmark_generate(target, target_tok, PROMPT)
r_speculative = benchmark_generate(target, target_tok, PROMPT, draft_model=draft)

print(f"Baseline:    {r_baseline['tok/s']:.1f} tok/s")
print(f"Speculative: {r_speculative['tok/s']:.1f} tok/s")
print(f"Speedup:     {r_speculative['tok/s'] / r_baseline['tok/s']:.2f}×")
