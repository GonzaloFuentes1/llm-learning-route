GRPO y RL para razonamiento

Group Relative Policy Optimization: el algoritmo que está detrás de DeepSeek-R1 y los modelos de razonamiento modernos. Entrena el modelo con recompensas verificables (matemáticas, código) sin necesitar un reward model aprendido separado.

1. ¿Por qué GRPO en lugar de PPO?

PPO (RLHF clásico)GRPO (DeepSeek-R1)
Reward modelRM aprendido (otro LLM)Función determinista (regex, test)
BaselineValue network (otro modelo)Media del grupo de respuestas
ComplejidadAlta (3 modelos en memoria)Baja (1 modelo + referencia)
Reward hackingPosible si el RM se saturaNo posible con rewards verificables
Dominio idealAlineamiento generalMatemáticas, código, razonamiento formal

En GRPO, para cada prompt se generan G respuestas del modelo actual. La recompensa de cada respuesta se normaliza restando la media del grupo y dividiendo por la desviación estándar — este es el "baseline" que en PPO lo hace una value network. Resultado: mismo efecto de reducción de varianza pero sin modelo de valor.

Recompensas verificables: la clave de DeepSeek-R1 es que las tareas de matemáticas y código tienen respuesta objetivamente correcta o incorrecta. Esto elimina la necesidad de un RM aprendido que puede hackear.

2. Funciones de recompensa

Las reward functions toman el prompt + la respuesta generada y devuelven un escalar. Para razonamiento matemático suelen combinarse varias:

python — reward_functions.py
import re

# ── Reward 1: respuesta correcta ─────────────────────────────────────────────
def reward_correctness(completions: list[str], answers: list[str], **kwargs) -> list[float]:
    """Verifica si la respuesta final dentro de \\boxed{} es correcta."""
    rewards = []
    for completion, answer in zip(completions, answers):
        # Extraer número dentro de \boxed{...}
        match = re.search(r"\\boxed\{([^}]+)\}", completion)
        if match:
            predicted = match.group(1).strip().replace(",", "")
            expected  = str(answer).strip().replace(",", "")
            rewards.append(2.0 if predicted == expected else 0.0)
        else:
            rewards.append(0.0)   # no hay respuesta formateada
    return rewards


# ── Reward 2: formato correcto ────────────────────────────────────────────────
def reward_format(completions: list[str], **kwargs) -> list[float]:
    """Premio si el modelo usa el formato ... y \\boxed{}."""
    rewards = []
    for completion in completions:
        has_think = bool(re.search(r".*?", completion, re.DOTALL))
        has_boxed = bool(re.search(r"\\boxed\{", completion))
        rewards.append(0.5 if (has_think and has_boxed) else 0.0)
    return rewards


# ── Reward 3: longitud razonable (anti-padding) ───────────────────────────────
def reward_length(completions: list[str], **kwargs) -> list[float]:
    """Penaliza respuestas muy cortas o excesivamente largas."""
    rewards = []
    for completion in completions:
        n = len(completion.split())
        if 50 <= n <= 800:
            rewards.append(0.1)
        elif n < 20 or n > 2000:
            rewards.append(-0.5)
        else:
            rewards.append(0.0)
    return rewards


# Combinar todas las reward functions
REWARD_FUNCTIONS = [reward_correctness, reward_format, reward_length]

3. Preparar el dataset

GRPO no necesita respuestas en el dataset de entrenamiento — solo prompts y la respuesta correcta para verificar. El modelo genera sus propias respuestas.

python — prepare_dataset.py
from datasets import load_dataset

# GSM8K en español (o cualquier dataset de matemáticas)
dataset = load_dataset("openai/gsm8k", "main", split="train")

SYSTEM_PROMPT = """Eres un asistente matemático. Razona paso a paso dentro de etiquetas  y ,
luego da tu respuesta final dentro de \\boxed{}.

Ejemplo:

Primero calculo...
Luego...

La respuesta es \\boxed{42}."""

def format_prompt(example):
    return {
        "prompt": [
            {"role": "system",  "content": SYSTEM_PROMPT},
            {"role": "user",    "content": example["question"]},
        ],
        "answer": example["answer"].split("####")[-1].strip(),   # número final
    }

dataset = dataset.map(format_prompt, remove_columns=dataset.column_names)
print(dataset[0])

4. Entrenamiento con GRPOTrainer

python — grpo_train.py
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig
from trl import GRPOTrainer, GRPOConfig
from datasets import load_dataset

MODEL = "Qwen/Qwen2.5-3B-Instruct"   # modelo base — necesita capacidad básica de razonamiento
OUTPUT = "models/grpo-math-es"

tokenizer = AutoTokenizer.from_pretrained(MODEL)
model = AutoModelForCausalLM.from_pretrained(
    MODEL,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)

dataset = load_dataset("json", data_files="data/gsm8k_es_prompts.jsonl", split="train")

# Importar reward functions del archivo anterior
from reward_functions import REWARD_FUNCTIONS

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
    bias="none",
)

training_args = GRPOConfig(
    output_dir=OUTPUT,
    num_train_epochs=2,
    learning_rate=1e-5,
    per_device_train_batch_size=1,
    gradient_accumulation_steps=16,
    num_generations=8,          # G: cuántas respuestas genera por prompt (el "grupo")
    max_prompt_length=512,
    max_completion_length=1024, # espacio para el chain-of-thought
    beta=0.04,                  # KL penalty contra el modelo de referencia
    temperature=0.9,            # temperatura de sampling — más alta = más diversidad
    bf16=True,
    logging_steps=5,
    save_steps=100,
    report_to="wandb",
)

trainer = GRPOTrainer(
    model=model,
    args=training_args,
    train_dataset=dataset,
    reward_funcs=REWARD_FUNCTIONS,  # lista de funciones; los scores se suman
    peft_config=lora_config,
)

trainer.train()
trainer.save_model(OUTPUT)
Hiperparámetros clave:
  • num_generations (G): más generaciones = baseline más estable pero más VRAM. Empieza con 8.
  • beta: KL penalty. Muy alto = el modelo no se aleja del SFT. Muy bajo = olvida el formato de chat.
  • temperature: necesitas diversidad en el grupo. Si todas las respuestas son idénticas, el gradiente es 0.

5. Métricas durante el entrenamiento

Métrica en W&BQué mideSeñal saludable
reward/meanRecompensa promedio del grupoSube gradualmente
reward/stdDiversidad de rewards en el grupoNo colapsa a 0 (necesitas varianza)
klDivergencia KL vs modelo de referenciaSe estabiliza; no explota
completion_lengthLongitud de las respuestas generadasSube (el modelo aprende a razonar más)
reward_correctness/mean% de respuestas correctas en el grupoSube de ~10% a 50%+
Colapso de reward: si reward/std cae a 0, el modelo genera respuestas idénticas — no hay señal de gradiente. Sube la temperatura de sampling o usa más diversas seed prompts.

6. Rewards verificables para código

python — code_reward.py
import re
import subprocess
import tempfile
import os

def reward_code_execution(completions: list[str], test_cases: list[str], **kwargs) -> list[float]:
    """Ejecuta el código generado contra casos de prueba en un proceso aislado."""
    rewards = []
    for completion, tests in zip(completions, test_cases):
        # Extraer bloque de código Python
        match = re.search(r"```python\n(.*?)```", completion, re.DOTALL)
        if not match:
            rewards.append(-1.0)
            continue

        code = match.group(1)
        full_code = code + "\n\n" + tests   # agregar asserts

        try:
            with tempfile.NamedTemporaryFile(mode="w", suffix=".py", delete=False) as f:
                f.write(full_code)
                tmp_path = f.name

            result = subprocess.run(
                ["python", tmp_path],
                timeout=5,       # timeout por seguridad
                capture_output=True,
                text=True,
            )
            os.unlink(tmp_path)

            if result.returncode == 0:
                rewards.append(2.0)   # código correcto
            else:
                rewards.append(0.0)   # código con error en tests
        except subprocess.TimeoutExpired:
            rewards.append(-0.5)      # timeout
        except Exception:
            rewards.append(-1.0)

    return rewards
Seguridad: ejecutar código generado por LLMs en el mismo proceso es peligroso. En producción usa sandboxing (contenedores Docker, nsjail, o servicios como E2B). El ejemplo de arriba usa subprocess con timeout como primera línea de defensa.
