Serving con vLLM

Cómo levantar un servidor de inferencia de alto rendimiento con vLLM: instalación, lanzamiento, requests con la API OpenAI-compatible y monitoreo de throughput.

1. Por qué vLLM es más rápido

La generación autoregresiva naive mantiene el KV-cache completo en VRAM para cada request. Con muchos requests concurrentes, la memoria se fragmenta y la GPU queda subutilizada. vLLM resuelve esto con PagedAttention:

  • Gestiona el KV-cache como páginas de memoria virtual (similar a la memoria virtual del SO)
  • Elimina la fragmentación: múltiples requests comparten páginas cuando tienen el mismo prefijo
  • Continuous batching: procesa tokens de distintos requests en paralelo, sin esperar que uno termine
  • Resultado: 2-24x más throughput que HuggingFace naive generate()

2. Instalación

bash
# Requiere CUDA 12.1+ y GPU con compute capability >= 7.0 (Volta+)
pip install vllm

# Verificar instalación
python -c "from vllm import LLM; print('vLLM OK')"

# Con soporte para Flash Attention (recomendado para H100/A100)
pip install vllm --extra-index-url https://download.pytorch.org/whl/cu121

3. Lanzar el servidor

bash — servidor básico (1 GPU)
python -m vllm.entrypoints.openai.api_server \
  --model /path/to/latamgpt-7b \
  --max-model-len 4096 \
  --gpu-memory-utilization 0.90 \
  --port 8000
bash — servidor con tensor parallelism (2+ GPUs)
python -m vllm.entrypoints.openai.api_server \
  --model /path/to/latamgpt-70b \
  --tensor-parallel-size 4 \       # distribuir entre 4 GPUs
  --max-model-len 8192 \
  --gpu-memory-utilization 0.90 \
  --dtype bfloat16 \
  --host 0.0.0.0 \
  --port 8000 \
  --served-model-name latamgpt
Parámetros clave:
• --max-model-len: longitud máxima de contexto (prompt + generación). Reducirlo aumenta el throughput.
• --gpu-memory-utilization: fracción de VRAM para el KV-cache. 0.90 es un buen default.
• --tensor-parallel-size: para modelos que no caben en 1 GPU.

4. Hacer requests

vLLM expone una API compatible con OpenAI, por lo que puedes usar cualquier cliente OpenAI.

Con curl
bash
# Completions (modelo base)
curl http://localhost:8000/v1/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "latamgpt",
    "prompt": "El procesamiento del lenguaje natural en Latinoamérica",
    "max_tokens": 200,
    "temperature": 0.7,
    "top_p": 0.9
  }'

# Chat completions (modelo instruct)
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "latamgpt",
    "messages": [{"role": "user", "content": "Explica qué es el fine-tuning"}],
    "max_tokens": 300,
    "temperature": 0.8
  }'
Con Python (cliente OpenAI)
python — cliente.py
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="token-no-necesario",  # vLLM no requiere API key por defecto
)

# Completions
response = client.completions.create(
    model="latamgpt",
    prompt="Los modelos de lenguaje para español deben",
    max_tokens=150,
    temperature=0.7,
    top_p=0.95,
    stop=["\n\n"],
)
print(response.choices[0].text)

# Chat (con streaming)
stream = client.chat.completions.create(
    model="latamgpt",
    messages=[
        {"role": "system", "content": "Eres un asistente en español."},
        {"role": "user", "content": "¿Qué es GRPO?"},
    ],
    max_tokens=300,
    stream=True,
)
for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)

5. Monitorear throughput y GPU

bash
# Ver métricas del servidor vLLM (en otra terminal)
curl http://localhost:8000/metrics

# Monitorear GPU mientras el servidor está corriendo
watch -n 2 nvidia-smi

# Benchmark simple con ab (Apache Bench)
ab -n 100 -c 10 -p payload.json -T application/json \
   http://localhost:8000/v1/completions
python — benchmark_throughput.py
import asyncio
import time
from openai import AsyncOpenAI

client = AsyncOpenAI(base_url="http://localhost:8000/v1", api_key="none")

async def single_request(prompt: str) -> dict:
    start = time.perf_counter()
    resp = await client.completions.create(
        model="latamgpt", prompt=prompt, max_tokens=100
    )
    elapsed = time.perf_counter() - start
    tokens = resp.usage.completion_tokens
    return {"tokens": tokens, "latency_s": elapsed, "tps": tokens / elapsed}


async def benchmark(prompts: list[str], concurrency: int = 10):
    results = await asyncio.gather(*[single_request(p) for p in prompts[:concurrency]])
    avg_tps = sum(r["tps"] for r in results) / len(results)
    avg_lat = sum(r["latency_s"] for r in results) / len(results)
    total_tokens = sum(r["tokens"] for r in results)
    print(f"Concurrency={concurrency} | Avg TPS={avg_tps:.1f} | Avg latency={avg_lat:.2f}s | Total tokens={total_tokens}")


prompts = ["La inteligencia artificial en Latam"] * 50
asyncio.run(benchmark(prompts, concurrency=10))
