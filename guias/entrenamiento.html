Código de Entrenamiento

Loop de pretraining con HuggingFace Accelerate y DeepSpeed ZeRO. Cubre configuración multi-GPU, gradient accumulation, precisión mixta bf16, checkpointing y cómo interpretar la loss curve.

1. Configurar Accelerate

Accelerate abstrae la complejidad de multi-GPU, DeepSpeed y FSDP. Una sola línea de config determina cómo se distribuye el entrenamiento.

bash
pip install accelerate deepspeed

# Generar config interactivamente (responde las preguntas)
accelerate config --config_file configs/deepspeed_zero2.yaml

# O crear el archivo directamente:
yaml — configs/deepspeed_zero2.yaml
compute_environment: LOCAL_MACHINE
distributed_type: DEEPSPEED
deepspeed_config:
  deepspeed_multinode_launcher: standard
  gradient_accumulation_steps: 4
  gradient_clipping: 1.0
  offload_optimizer_device: none
  offload_param_device: none
  zero3_init_flag: false
  zero_stage: 2
machine_rank: 0
main_training_function: main
mixed_precision: bf16
num_machines: 1
num_processes: 4          # = número de GPUs
use_cpu: false

2. Training loop básico

python — train.py
import math
import torch
from torch.utils.data import DataLoader
from transformers import AutoModelForCausalLM, AutoTokenizer, get_cosine_schedule_with_warmup
from accelerate import Accelerator
import wandb


def main():
    accelerator = Accelerator(
        gradient_accumulation_steps=4,
        mixed_precision="bf16",
        log_with="wandb",
    )

    # Inicializar W&B solo en el proceso principal
    if accelerator.is_main_process:
        wandb.init(project="latamgpt", name="pretraining-v1")

    # Cargar modelo y tokenizador
    model = AutoModelForCausalLM.from_pretrained(
        "meta-llama/Llama-2-7b-hf",
        torch_dtype=torch.bfloat16,
        attn_implementation="flash_attention_2",  # requiere flash-attn
    )

    tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")

    # Optimizer con weight decay selectivo (no aplicar a bias y layernorm)
    decay_params = [p for n, p in model.named_parameters() if "bias" not in n and "norm" not in n]
    no_decay_params = [p for n, p in model.named_parameters() if "bias" in n or "norm" in n]

    optimizer = torch.optim.AdamW(
        [{"params": decay_params, "weight_decay": 0.1},
         {"params": no_decay_params, "weight_decay": 0.0}],
        lr=3e-4,
        betas=(0.9, 0.95),
        eps=1e-8,
    )

    # Dataloader (tu dataset tokenizado)
    train_dataloader = DataLoader(dataset, batch_size=4, shuffle=True)

    # LR scheduler: warmup + cosine decay
    num_training_steps = len(train_dataloader) * NUM_EPOCHS
    num_warmup_steps = int(0.01 * num_training_steps)  # 1% de warmup
    scheduler = get_cosine_schedule_with_warmup(optimizer, num_warmup_steps, num_training_steps)

    # Preparar para distribución (Accelerate maneja DDP / DeepSpeed)
    model, optimizer, train_dataloader, scheduler = accelerator.prepare(
        model, optimizer, train_dataloader, scheduler
    )

    global_step = 0
    model.train()

    for epoch in range(NUM_EPOCHS):
        for step, batch in enumerate(train_dataloader):
            with accelerator.accumulate(model):  # gradient accumulation
                input_ids = batch["input_ids"]
                labels = batch["labels"]

                outputs = model(input_ids=input_ids, labels=labels)
                loss = outputs.loss

                accelerator.backward(loss)

                # Gradient clipping (solo aplica al paso real, no acumulación)
                if accelerator.sync_gradients:
                    grad_norm = accelerator.clip_grad_norm_(model.parameters(), max_norm=1.0)

                optimizer.step()
                scheduler.step()
                optimizer.zero_grad()

            # Logging
            if global_step % 10 == 0 and accelerator.is_main_process:
                wandb.log({
                    "train/loss": loss.item(),
                    "train/perplexity": math.exp(loss.item()),
                    "train/lr": scheduler.get_last_lr()[0],
                    "train/grad_norm": grad_norm.item() if accelerator.sync_gradients else None,
                    "train/step": global_step,
                })

            # Checkpoint cada 1000 pasos
            if global_step % 1000 == 0 and global_step > 0:
                if accelerator.is_main_process:
                    unwrapped = accelerator.unwrap_model(model)
                    unwrapped.save_pretrained(f"checkpoints/step_{global_step}")
                    tokenizer.save_pretrained(f"checkpoints/step_{global_step}")

            global_step += 1


if __name__ == "__main__":
    main()

3. Lanzar el entrenamiento

bash
# Con Accelerate (recomendado)
accelerate launch \
  --config_file configs/deepspeed_zero2.yaml \
  --num_processes 4 \
  train.py

# Con torchrun (equivalente manual)
torchrun --nproc_per_node=4 train.py

# Multi-nodo (2 nodos x 4 GPUs = 8 GPUs)
# En nodo 0:
accelerate launch \
  --config_file configs/deepspeed_zero2.yaml \
  --num_processes 8 \
  --num_machines 2 \
  --machine_rank 0 \
  --main_process_ip 10.0.0.1 \
  --main_process_port 29500 \
  train.py

5. Interpretar la loss curve

La forma de la loss curve durante pretraining es una señal diagnóstica crítica. Aquí lo que esperar:

text — fases típicas del training
Fase 1 — Warmup (primeros ~500 steps):
  Loss: baja rápidamente de ~12-14 a ~3-5
  LR: sube linealmente hasta el peak (ej: 3e-4)
  Acción: normal, no intervenir

Fase 2 — Descenso estable (bulk del training):
  Loss: baja gradualmente, ~0.001-0.01 por 1000 steps
  LR: decae con cosine schedule
  Gradient norm: estable, tipicamente 0.5-2.0
  Acción: monitorear, hacer checkpoints

Fase 3 — Loss spike (señal de alerta):
  Loss: sube bruscamente +0.3 o más en pocos steps
  Grad norm: puede subir a 10-100x lo normal
  Causas: batch con texto corrupto, LR muy alto, overflow en bf16
  Acción: ver debugging.html

Fase 4 — Cooldown (últimos ~5-10% del training):
  Loss: baja más rápido al reducir LR al mínimo
  Acción: no interrumpir, es cuando el modelo "consolida"
Perplexity como métrica: PPL = exp(loss). Un buen modelo de lenguaje en español alcanza PPL ≈ 8-15 en el set de validación. PPL > 50 sugiere problemas en los datos o configuración.
