FineWeb-Edu: etiquetar con LLM y entrenar un clasificador BERT
La misma estrategia de HuggingFace para construir FineWeb-Edu, adaptada a español y portugués: pedir a un LLM que puntúe páginas, entrenar un clasificador BERT sobre esas etiquetas y correr inferencia masiva sobre el corpus completo.
1. La estrategia completa
~500k páginas web
→
LLM anota calidad (0-5)
→
Entrenar BERT regresor
→
Inferencia en billones de docs
→
Filtrar score ≥ 3

El problema: un LLM grande (Llama 3 70B) puede puntuar la calidad educativa de un texto con alta precisión, pero es demasiado lento y caro para procesar billones de páginas de CommonCrawl. La solución de FineWeb-Edu: usar el LLM solo sobre una muestra (~500k docs), entrenar un BERT pequeño en esas etiquetas, y correr el BERT —que es 1000× más rápido— sobre el corpus completo.

EtapaModeloVelocidadCosto
AnotaciónLlama 3 70B Instruct~5 docs/s (GPU cluster)Alto — solo 500k docs
Entrenamiento BERTDistilBERT / XLM-R-base1-4 horas (A100)Bajo — una sola vez
Inferencia masivaBERT fine-tuned~5,000 docs/s (A100)Bajo — escala lineal
2. Prompt de anotación con el LLM

El prompt tiene que ser concreto y forzar una respuesta numérica parseable. La escala 0-5 permite al modelo expresar diferencias sutiles sin generar desacuerdo entre anotadores.

python
ANNOTATION_PROMPT = """A continuación se muestra un extracto de una página web en español o portugués.
Evalúa su valor educativo usando la siguiente escala:

0 - Sin valor educativo: spam, publicidad, contenido generado automáticamente, listas de productos.
1 - Poco valor: algo de información pero superficial o irrelevante para el aprendizaje.
2 - Valor moderado: tiene contenido informativo pero mezclado con mucho ruido o sin estructura clara.
3 - Buen valor educativo: texto informativo, bien escrito, útil para aprender un tema.
4 - Alto valor educativo: texto claro, estructurado, con explicaciones, ejemplos o definiciones.
5 - Valor educativo excepcional: material de estudio, libro de texto, explicación detallada con rigor.

Extracto:
\"\"\"
{text}
\"\"\"

Responde ÚNICAMENTE con un número entero del 0 al 5. No agregues explicación."""


def build_annotation_messages(text: str) -> list[dict]:
    return [
        {"role": "system",  "content": "Eres un evaluador experto de calidad educativa de textos en español y portugués. Solo respondes con un número del 0 al 5."},
        {"role": "user",    "content": ANNOTATION_PROMPT.format(text=text[:2000])},
    ]
Anotar en batch con vLLM (recomendado para volumen)
python
import re
from vllm import LLM, SamplingParams
from transformers import AutoTokenizer
from datasets import load_dataset, Dataset

ANNOTATOR_MODEL = "meta-llama/Meta-Llama-3.1-70B-Instruct"

llm = LLM(
    model=ANNOTATOR_MODEL,
    tensor_parallel_size=4,       # 70B requiere al menos 4× A100 80GB
    gpu_memory_utilization=0.90,
    max_model_len=4096,
)
tok = AutoTokenizer.from_pretrained(ANNOTATOR_MODEL)

sampling = SamplingParams(
    temperature=0.0,      # determinista — queremos consistencia
    max_tokens=4,         # solo necesitamos un dígito
    stop=["\n", " "],
)

def parse_score(text: str) -> int | None:
    """Extrae el entero 0-5 de la respuesta del LLM."""
    m = re.search(r"[0-5]", text.strip())
    return int(m.group()) if m else None


def annotate_batch(texts: list[str]) -> list[int | None]:
    prompts = [
        tok.apply_chat_template(
            build_annotation_messages(t),
            add_generation_prompt=True,
            tokenize=False,
        )
        for t in texts
    ]
    outputs = llm.generate(prompts, sampling)
    return [parse_score(o.outputs[0].text) for o in outputs]


# Cargar corpus y anotar
raw_ds = load_dataset("uonlp/CulturaX", "es", split="train", streaming=True)
sample = list(raw_ds.take(500_000))   # muestra de anotación

BATCH = 512
scores, valid_texts = [], []

for i in range(0, len(sample), BATCH):
    batch_texts = [ex["text"] for ex in sample[i:i+BATCH]]
    batch_scores = annotate_batch(batch_texts)
    for text, score in zip(batch_texts, batch_scores):
        if score is not None:
            scores.append(score)
            valid_texts.append(text)
    if i % 10_000 == 0:
        print(f"Anotados: {i:,} | válidos: {len(valid_texts):,}")

annotation_ds = Dataset.from_dict({"text": valid_texts, "score": scores})
annotation_ds.save_to_disk("annotation_dataset_es")
Presupuesto reducido: Si no tienes acceso a Llama 70B, usa Llama 3.1 8B Instruct o un modelo de API. La calidad de las etiquetas cae un poco pero el clasificador BERT final sigue siendo útil. Con 50k docs ya puedes entrenar un clasificador razonable.

4. Entrenar el clasificador BERT

Entrenamos un modelo de regresión: el BERT produce un score continuo 0-5 que luego usamos para filtrar. Usamos MSE loss sobre los scores del LLM anotador. Para es/pt usamos XLM-RoBERTa-base como backbone — fue entrenado sobre 100 idiomas y es la mejor opción multilingüe.

python
from datasets import load_from_disk, DatasetDict
from transformers import (
    AutoTokenizer, AutoModelForSequenceClassification,
    TrainingArguments, Trainer,
)
import torch
import numpy as np
from sklearn.metrics import mean_squared_error, mean_absolute_error

BACKBONE = "xlm-roberta-base"      # multilingüe es/pt/en/...
# Alternativas monolingüe: "dccuchile/bert-base-spanish-wwm-cased" (BETO)
#                          "neuralmind/bert-base-portuguese-cased" (BERTimbau)

tokenizer = AutoTokenizer.from_pretrained(BACKBONE)

# Preparar dataset
ds = load_from_disk("annotation_dataset_es")
split = ds.train_test_split(test_size=0.05, seed=42)

def tokenize(batch):
    enc = tokenizer(
        batch["text"],
        truncation=True,
        max_length=512,
        padding="max_length",
    )
    # El score es float para regresión
    enc["labels"] = [float(s) for s in batch["score"]]
    return enc

tokenized = DatasetDict({
    "train": split["train"].map(tokenize, batched=True, remove_columns=["text","score"]),
    "test":  split["test"].map(tokenize,  batched=True, remove_columns=["text","score"]),
})
tokenized.set_format("torch")

# Modelo: num_labels=1 → regresión
model = AutoModelForSequenceClassification.from_pretrained(
    BACKBONE, num_labels=1
)

def compute_metrics(eval_pred):
    preds, labels = eval_pred
    preds = preds.squeeze()
    return {
        "mse": mean_squared_error(labels, preds),
        "mae": mean_absolute_error(labels, preds),
        "rmse": np.sqrt(mean_squared_error(labels, preds)),
    }

training_args = TrainingArguments(
    output_dir="edu-classifier-xlmr",
    num_train_epochs=3,
    per_device_train_batch_size=32,
    per_device_eval_batch_size=64,
    learning_rate=2e-5,
    lr_scheduler_type="cosine",
    warmup_ratio=0.06,
    bf16=True,
    eval_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    metric_for_best_model="mse",
    greater_is_better=False,
    logging_steps=100,
    dataloader_num_workers=4,
    report_to="wandb",
    run_name="edu-classifier-xlmr-v1",
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized["train"],
    eval_dataset=tokenized["test"],
    compute_metrics=compute_metrics,
)

trainer.train()
trainer.save_model("edu-classifier-xlmr-best")
tokenizer.save_pretrained("edu-classifier-xlmr-best")
Métricas objetivo: Con 500k ejemplos de entrenamiento, XLM-R-base suele alcanzar MAE ≈ 0.55 y RMSE ≈ 0.72. Para el filtrado de datos lo que importa es que el clasificador acierte en qué textos están por encima o por debajo del umbral, no la precisión decimal del score.
5. Calibrar el umbral de filtrado

El umbral óptimo depende de cuánto corpus queremos retener vs. la calidad deseada. FineWeb-Edu usó score ≥ 3 y retuvo ~9% del corpus original.

python
import numpy as np
from transformers import pipeline

classifier = pipeline(
    "text-classification",
    model="edu-classifier-xlmr-best",
    tokenizer=tokenizer,
    device=0,
    function_to_apply="none",  # devuelve el logit crudo (= el score)
)

# Evaluar sobre el test set para elegir umbral
test_texts  = split["test"]["text"][:5000]
test_scores = split["test"]["score"][:5000]

pred_scores = [r["score"] for r in classifier(test_texts, batch_size=128, truncation=True)]

# Para cada umbral: ¿cuánto corpus retenemos y qué calidad media tiene?
print(f"{'Umbral':<10} {'% retenido':<15} {'Precision@3+':<15}")
for threshold in [2.0, 2.5, 3.0, 3.5, 4.0]:
    kept  = [p >= threshold for p in pred_scores]
    true_qual = [s >= 3 for p, s in zip(pred_scores, test_scores) if p >= threshold]
    pct_kept  = sum(kept) / len(kept) * 100
    precision = sum(true_qual) / len(true_qual) * 100 if true_qual else 0
    print(f"{threshold:<10.1f} {pct_kept:<15.1f} {precision:<15.1f}")

6. Inferencia masiva sobre el corpus completo

Con el clasificador entrenado, lo corremos sobre el corpus completo. El patrón es el mismo que la guía de inferencia con BERT, pero aquí el objetivo es filtrar billones de documentos, así que la eficiencia importa.

python
import torch
from torch.utils.data import DataLoader, Dataset as TorchDataset
from transformers import AutoTokenizer, AutoModelForSequenceClassification
from datasets import load_dataset
import numpy as np

THRESHOLD = 3.0
CLASSIFIER_PATH = "edu-classifier-xlmr-best"
BATCH_SIZE = 512
MAX_LENGTH = 512

tok_inf = AutoTokenizer.from_pretrained(CLASSIFIER_PATH)
clf = AutoModelForSequenceClassification.from_pretrained(
    CLASSIFIER_PATH, torch_dtype=torch.bfloat16
).to("cuda").eval()


class TextDataset(TorchDataset):
    def __init__(self, texts):
        self.texts = texts

    def __len__(self):
        return len(self.texts)

    def __getitem__(self, idx):
        return self.texts[idx]


def collate_fn(batch):
    return tok_inf(
        batch,
        max_length=MAX_LENGTH,
        truncation=True,
        padding=True,
        return_tensors="pt",
    )


@torch.no_grad()
def score_texts(texts: list[str]) -> np.ndarray:
    ds = TextDataset(texts)
    loader = DataLoader(ds, batch_size=BATCH_SIZE, collate_fn=collate_fn,
                        num_workers=4, pin_memory=True)
    all_scores = []
    for batch in loader:
        batch = {k: v.to("cuda") for k, v in batch.items()}
        logits = clf(**batch).logits.squeeze(-1)
        all_scores.append(logits.float().cpu().numpy())
    return np.concatenate(all_scores)


# Procesar el corpus completo en shards para no cargar todo en RAM
corpus = load_dataset("uonlp/CulturaX", "es", split="train", streaming=True)

kept_docs = []
shard_size = 50_000
shard = []
total_seen, total_kept = 0, 0

for doc in corpus:
    shard.append(doc["text"])
    if len(shard) == shard_size:
        scores = score_texts(shard)
        mask = scores >= THRESHOLD
        kept_docs.extend([t for t, m in zip(shard, mask) if m])
        total_seen += shard_size
        total_kept += int(mask.sum())
        print(f"Procesados: {total_seen:,} | Retenidos: {total_kept:,} "
              f"({total_kept/total_seen*100:.1f}%)")
        shard = []

# Guardar resultado
from datasets import Dataset
filtered = Dataset.from_dict({"text": kept_docs})
filtered.push_to_hub("latamgpt/culturax-es-edu-filtered", private=True)
Throughput esperado: XLM-R-base con batch=512 y bf16 en una A100 procesa ~5,000 documentos/segundo. Un shard de 50M documentos tarda ~2.8 horas. Para el corpus completo de CulturaX en español (~200M docs), ~12 horas en una A100.
